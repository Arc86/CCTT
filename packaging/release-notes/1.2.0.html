CCTT 1.2.0

  • See when your 5-hour limit resets — even without live limits. CCTT now works out your 5-hour window from your own session history, so the popover shows "resets in 4 hours" whether or not you've turned live limits on. Previously that countdown only existed on the live path.
  • Know if you'll run out before the reset. When you're burning through a window faster than it can sustain, the gauge now says so — "At this pace: limit reached 16:10" — so you find out with time to react, instead of at 100%.
  • The 5-hour number now resets instead of drifting down. Claude's 5-hour window starts at your first message and runs a fixed five hours; going idle doesn't reset it. CCTT now measures it that way, so your 5-hour usage drops to 0% the moment a window closes rather than slowly decaying. This is the number behaving correctly — but it will look different from what you're used to.
  • Honest live status. If your connection is rate-limited or your login needs refreshing, the badge now says rate-limited or reconnect instead of quietly showing an ever-older reading labelled "Live". CCTT also backs off politely when rate-limited, and recovers on its own.
  • Publish your usage for other tools (optional). Turn on Settings ▸ Data ▸ Publish usage.json and CCTT writes a small status file to ~/.cctt/usage.json — handy for a Claude Code statusline or your own dashboard. Off by default. CCTT still only ever reads Claude Code's data, never writes to it.
  • VoiceOver fix. The menu-bar item no longer reads its percentage twice.
